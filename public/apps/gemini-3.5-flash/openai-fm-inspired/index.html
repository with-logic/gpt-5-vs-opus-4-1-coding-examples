SYSTEM ARCHITECTURE SPECIFICATIONS

1. Web Audio Procedural Synthesis:
Unlike standard media streaming players, OpenAI.fm synthesizes all audio on-the-fly inside your browser. No external MP3 files are used. The drum machines, retro-synthesizer pads, lead melodies, and background vinyl crackle are physical sound wave oscillators (sine, square, sawtooth, triangle waves) computed in real-time.

2. Precise Temporal Scheduling:
Implements an ultra-precise dual-thread audio scheduler clock (lookahead queues running at 25ms intervals), which schedules oscillator and envelope node parameters using the high-resolution hardware timeline of the audio context to eliminate rhythm jitter.

3. Real-Time FFT Analyser:
Connected before the auditory output, a fast Fourier transform node retrieves audio spectrum frequency values (2048 bins) and waveform time-domain sequences. This data is physically analyzed on every animation frame to feed our Canvas visualizers.

4. Chromatic Aberration Visualizers:
Canvas frames utilize additive rendering pipelines, radial glowing particles, neural connectivity lines that burst with synapse photon bubbles, and projected 3D perspective scroll grids.

5. Real-Time AI DJ Speech & Autoducking:
Utilizes the Web Speech Synthesis API. Features a fully custom auto-ducking controller that dynamically scales the background instrument gain node to 20% when the DJ begins speaking and ramps it back smoothly after voice transmissions terminate.
